For-of

For-of

Як 20 років назад... 
				
				for (var index = 0; index < myArray.length; index++) {
					console.log(myArray[index]);
				}

For-of

Як в ES5 
				
				myArray.forEach(function (value) {
					console.log(value);
				});

For-of

А як на рахунок For-in? 
				
				for (var index in myArray) {
					console.log(myArray[index]);
				}
    але так робити не варто...
  • рядкова арифметика (index - рядок)
  • ймовірність проходу лишній раз по елементам
  • прохід у випадковому порядку! О_о

For-of

Нас врятує! 
				
				for (var index of myArray) {
					console.log(myArray[index]);
				}
    Виглядає непогано
  • лаконичній та наглядний синтаксис
  • відсутні недоліки for-in
  • на відміну від forEach(), працює із break, continue та return

For-of

Підсумок
  • for-in потрібний для перебору властивостей об'єкта.
  • for-of потрібний для перебору даних, наприклад, значень масива
  • for-of також добре працює із більшістю масивоподібних об'єктів, таких як NodeList в DOM
  • працює із Map та Set
  • працює із рядками, розглядаючи рядок як набір символів Unicode

"Генератори - співпрограми першого класу, що представляють із себе об'єкти, інкапсулюючі відкладені контексти виконання"

Щооо?

Generators *

				function* compute(degree, max) {
					var a = 1;
					for (let i = 0; i < max; i++) {
						yield a;
						a = a * degree;
					}
				}
				let gen = compute(2, 3);
				gen.next(); // { value: 1, done: false }
				gen.next(); // { value: 2, done: false }
				gen.next(); // { value: 4, done: false }
				gen.next(); // { value: undefined, done: true }

Generators *

				function* seq(a) {	
					yield a + 5;	
					let b = yield null;	
					return a + b;	
				}	
				let gen = seq(5);	
				gen.next().value; // 10	
				gen.next().value; // null	
				gen.next(8).value; // 13

Callback -> Promise

			new Promise(function (fulfill, reject) {
				phone.callACat(function (error, result) {
					if (error) {
						reject(error);
						return;
					}
					fulfill(result);
				});
			});

Chaining

			phone.callACat()
			 .then(function (catResult) {
			   return phone.callADog(); // Promise
			 })
			 .then(function (dogResult) {
			   return 'Hello, Dog!'; // Value
			 })
			 .then(function (msg) { 
			   console.log(msg); // Hello, Dog! 
			 });

Обробка помилок

Модулі

CommonJS (CJS)

  • призначений для синхронної загрузки;
  • переважно використовується на стороні сервера;

Asynchronous Module Definition (AMD)

  • призначений для асинхронної загрузки;
  • переважно використовується на стороні клієнта;

Модулі

				// utility.js
				function generateRandom() {
					return Math.random();
				}
				function sum(a, b) {
					return a + b;
				}
				export { generateRandom, sum }
				export {generateRandom as random, sum as doSum}

Модулі

				// app.js
				import { generateRandom, sum } from 'utility';
				generateRandom() // random number
				sum(1, 2) // 3
